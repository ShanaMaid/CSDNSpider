前几天重装了系统后安装MySQL后发现按照最开始的安装方式会失败。

最开始安装MySQL配置了环境变量后都是打开cmd，依次输入以下命令：

1、mysqld -install   //安装Mysql，添加mysql服务

2、net start mysql  //启动mysql

3、mysql   //默认以root用户进入mysql,都知道刚安装好的mysql的root用户是没有密码的


但是这种安装办法在mysql官网下载的最新版本的mysql安装会失败，为什么呢？

因为mysql版本更新后在5.6后的某个版本删除了压缩文件里的data文件夹，具体原因不是很清楚。这就导致了我们依次执行上述命令会导致mysql无法启动以及进入。

按照官方文档的说明，我们应该这样进行安装。

还是先打开cmd,依次输入以下指令：

1、mysqld -initialize //初始化mysql，这会在根目录生成data文件夹，过程会有点长，稍微等待一下。

2、mysqld -install   //安装Mysql，添加mysql服务

3、net start mysql  //启动mysql

4、mysql   //默认以root用户进入mysql,都知道刚安装好的mysql的root用户是没有密码的



OK，这样就没有问题了~~~~！

最后这次得出的经验就是惯用的方法失效的时候，各位还是去看看官方文档吧。
