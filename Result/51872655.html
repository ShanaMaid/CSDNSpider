1、什么是Git?(取自度娘)

   Git是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

2、需要掌握的基本指令

   git remote add  origin   //添加远程仓库

   git init //初始化本地文件夹

   git add  //添加变更内容

   git commit  //为变更内容添加注释

   git pull //从远程仓库拉取文件更新

   git push //从本地仓库推送变动文件至远程仓库

4、本地仓库的初始化

首先新建一个文件夹，文件夹名字随意，可以与远程仓库名字不一样。

eg:yl_project

进入yl_project文件夹，在这个空文件夹目录下右键，选择Git Bash Here。

有的人可能安装了Git后可能直接右键没有这个选项，可以shift+右键 试试看。

选择Git Bash Here后会弹出类似cmd的窗口，这就是Git。


细心的人会发现还有一个Git GUI Here，这个是图形化界面，不推荐使用。

 

继续回到正题，在弹出的Git窗口输入git init,初始化本地仓库

初始化完成后会有

Initialized empty Git repository in C:/Users/Administrator/Desktop/yl_project/.git/

in后面的是路径名，因人而异。

5、关于git add ,git commit,git push

git add和git commit是一对好兄弟，必定成对出现，

也就是说你使用了git add后必须使用git commit。

 

git add用于提交变更的文件，git commit则是用于标注本次提交的变动，可以理解 为注释

一般用 git add -A     A=all，即提交本次全部变动

Git  commit -m “XXXXXX”   m=message  红色的XXX就是注释，即变动信息

 

Git add后记得commit

Eg:

Git add -A

Git commit -m “测试”

Git push origin master   //会弹出窗口叫你输入账户密码

 

 

 

6、关于文件冲突

比如一个test.txt文件里的内容是

I am good!

 

A把good修改成了bad,推到了服务器上，

而B在本地把good修改成了better,同时已经Git add和git commit了，此时B从服务器上拉取。那么test.txt就会产生冲突了，在冲突的地方会出现

 

 

<<<<<<< HEAD

I am bad!

=======

I am better!

>>>>>>> feature1

 

 

根据需要修改这段字符就可以了，比如最终决定保留的是 I am better!

 

 

<<<<<<< HEAD

I am bad!

=======

I am better!

>>>>>>> feature1

 

删除红色部分，然后git add   git commit即可，一般commit需要备注解决冲突。
